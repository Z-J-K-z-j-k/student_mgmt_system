# 快速远程测试指南

## 🚀 5分钟快速测试（两台机器）

### 机器A（服务器端）

1. **获取IP地址**
   ```bash
   python student_mgmt/server/test_server.py
   ```
   记录显示的IP地址，例如：`192.168.1.100`

2. **配置防火墙**（Windows）
   - 打开"Windows Defender 防火墙"
   - 高级设置 → 入站规则 → 新建规则
   - 端口 → TCP → 5000 → 允许连接

3. **启动服务器**
   ```bash
   python student_mgmt/server/app.py
   ```
   看到 `Running on http://0.0.0.0:5000` 表示成功

### 机器B（客户端）

1. **修改配置文件**
   编辑 `student_mgmt/client/config.py`：
   ```python
   SERVER_URL = "http://192.168.1.100:5000"  # 替换为机器A的IP
   ```

2. **测试连接**
   ```bash
   python student_mgmt/client/test_connection.py
   ```
   看到 `✅ 连接成功！` 表示配置正确

3. **启动客户端**
   ```bash
   python student_mgmt/client/main.py
   ```

4. **测试登录**
   - 输入用户名、密码、角色
   - 点击登录
   - 成功进入主界面即表示远程访问成功！

## 🔍 如果连接失败

### 检查清单

- [ ] 服务器是否正在运行？
- [ ] 服务器IP地址是否正确？
- [ ] 防火墙是否允许5000端口？
- [ ] 两台机器是否在同一网络？
- [ ] 能否ping通服务器IP？

### 快速诊断

**在客户端机器上运行：**
```bash
# 测试网络连通性
ping <服务器IP>

# 测试端口连通性（Windows需要启用Telnet客户端）
telnet <服务器IP> 5000
```

**如果ping不通：**
- 检查两台机器是否在同一WiFi/网络
- 检查服务器防火墙是否阻止了ICMP

**如果ping通但telnet不通：**
- 检查服务器防火墙是否允许5000端口
- 检查服务器是否真的在运行
